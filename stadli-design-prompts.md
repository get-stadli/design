# Stadli — Claude Design Meta Prompts (Consolidated)

A systems-first prompt library for Claude Design. Heavily consolidated from 250 single-asset prompts so that related templates are generated *together* from a shared foundation — which keeps everything visually consistent. Run a "system" prompt and you get the whole family in one matching pass, rather than building look-alike assets one at a time and watching them drift.

> **How to use:** Build the foundations (1–11) first, in the numbered step order — each step depends on the ones before it. Most later prompts reference the brand system, deck theme, or document template they inherit from, so once the foundations exist, everything downstream stays on-brand. Prompts 62–66 are wildcards that vary every run.

---

## Foundations

Build these in order. Each step depends on the steps before it, so finish a step completely before starting the next. Within a step, the prompts can be run in any order (or in parallel) because they only depend on earlier steps.

**Step 1 — Core identity (build this first, on its own; everything inherits from it):**

1. Build the complete Stadli brand system: logo lockups, WCAG-AA color palette, type scale, grid, and spacing tokens, as one cohesive foundation everything else inherits.

**Step 2 — Brand language (needs Step 1):**

2. Create the Stadli brand voice and copywriting guide: personality, tone rules, words to use and avoid, and before-and-after examples, aligned to the visual system.
3. Extend the Stadli brand system with a pattern, texture, and color-application set shown across three sample touchpoints, so the look holds together everywhere.

**Step 3 — Core asset systems (need Steps 1–2; can run in parallel):**

4. Design the Stadli icon and UI-component library: twenty-four feature and interface icons, button styles and states, and a favicon-scale symbol, all sharing one grid and weight.
5. Build the Stadli illustration system: style rules plus a modular kit of figures, objects, and backgrounds the team can recombine into endless on-brand scenes.
6. Create the Stadli photography and image art-direction system: style guide, color treatment and duotone, headshot direction, and a sourcing moodboard.

**Step 4 — System extensions (each needs its parent system from Step 3):**

7. Extend the Stadli illustration system into a product set: empty states, edge-case screens, loading frames, and a sticker and reaction pack, all in one style.
8. Build Stadli text-over-image and crop templates off the photo system: gradient scrims, type placement, and aspect-ratio adaptations for web, social, and print.
9. Build the Stadli motion and animated-brand kit: an animated logo, loading mark, video lower-thirds, and transition styles, all derived from the core identity.
10. Create the Stadli editorial and cover-art set: blog and article headers, abstract data-inspired report covers, and an illustrated vision timeline, in one consistent style.

**Step 5 — First applied artifact (needs the logo, color, type, and pattern from Steps 1–2):**

11. Design the Stadli print and stationery kit: business cards, letterhead, envelope, tri-fold brochure, and a flexible flyer-poster template, all unified.
